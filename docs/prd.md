\# Agolive PRD



\## 1. Problem



기존 협업 도구(Slack, Zoom, Discord 등)는 대화 중심 인터페이스로 구성되어 있어

사용자들이 “같은 공간에 존재한다”는 감각을 제공하지 못한다.



이로 인해:

\* 현재 누가 참여 중인지 직관적으로 파악하기 어렵고

\* 자연스러운 상호작용(옆에 가서 말 걸기)이 불가능하며

\* 커뮤니케이션이 의도된 회의 중심으로만 이루어진다



결과적으로 협업의 몰입도와 자발적인 상호작용이 감소한다.



\---



\## 2. Target User



\* 2\~10명 규모의 원격 협업 팀

\* Slack / Discord 기반 협업에 익숙한 사용자

\* 가벼운 상호작용이 많은 스타트업 / 스터디 그룹



\---



\## 3. Goals



\* 사용자들이 “같은 공간에 있다”는 느낌을 받도록 한다

\* 자연스러운 상호작용을 유도한다

\* 텍스트 중심 협업을 공간 기반 경험으로 확장한다



\---



\## 4. Overview



Agolive는 사용자가 가상의 공간에 아바타로 입장하여

다른 사용자와 실시간으로 위치를 공유하고 상호작용할 수 있는

공간 기반 협업 플랫폼이다.



\---



\## 5. User Flow



1\. 사용자는 방(Room)을 생성하거나 참여한다

2\. 아바타 형태로 공간에 입장한다

3\. 현재 접속 중인 사용자들을 시각적으로 확인한다

4\. 다른 사용자에게 이동한다

5\. 채팅을 통해 상호작용한다



\---



\## 6. Core Features (MVP)



\### Room Management



\* Public / Private 방 생성

\* 초대 링크 기반 접근

\* 방 목록 조회



\### Presence \& Movement



\* 사용자 위치(x, y) 실시간 동기화

\* 아바타 이동

\* 접속 상태 표시 (입장 / 퇴장)



\### Real-time Communication



\* 텍스트 채팅

\* 시스템 메시지 (입장, 퇴장 등)

\* 채팅 기록 저장



\---



\## 7. Technical Notes



\* 초기에는 단일 서버 기반 구조로 구현한다

\* 실시간 처리는 WebSocket 기반 이벤트 방식으로 구성한다

\* Room 단위로 상태를 관리하여 이후 확장 가능하도록 설계한다

\* 트래픽 증가 시 실시간 처리 영역을 별도 서비스로 분리할 수 있도록 고려한다



\---



\## 8. Roadmap



\### Phase 1 (MVP)



\- 방 생성 및 입장

\- 사용자 위치 실시간 공유

\- 텍스트 채팅

\- 기본적인 공간 기반 상호작용





\---



\### Phase 2: AI Agent



Goal:

\- 사용자 협업을 보조하는 단일 AI 에이전트 도입



User Experience:

\- 사용자는 방 안에서 AI 에이전트를 소환할 수 있다

\- 에이전트는 특정 역할(도우미, 요약자 등)을 가진다

\- 에이전트는 대화 맥락을 이해하고 응답한다

\- 에이전트의 응답은 실시간으로 표시된다





\---



\### Phase 3: Multi-Agent Collaboration



Goal:

\- 여러 에이전트가 협업하는 구조 도입



User Experience:

\- 여러 에이전트를 동시에 생성할 수 있다

\- 에이전트 간 대화 및 협업이 가능하다

\- 사용자는 특정 에이전트에게 역할을 할당할 수 있다

\- 필요 시 사용자 개입 (Human-in-the-loop)이 가능하다





\---



\### Phase 4: Tool Execution



Goal:

\- 에이전트가 실제 작업을 수행할 수 있도록 확장



User Experience:

\- 에이전트가 파일(문서, Excel 등)을 생성한다

\- 외부 도구 실행 결과를 사용자에게 제공한다

\- 결과물은 다운로드하거나 공유할 수 있다





\---



\### Phase 5: Voice \& Presence Enhancement



Goal:

\- 공간 기반 상호작용을 더욱 현실감 있게 확장



User Experience:

\- 음성 채팅 지원

\- 거리 기반 음성 볼륨 조절 (proximity audio)

\- 공간 몰입감 향상
